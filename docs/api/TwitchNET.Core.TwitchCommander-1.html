Class TwitchCommander<TCommander>

The TwitchCommander<TCommander> defines a framework for our module framework. The Commander tasks are to receive incoming messages from TwitchClient and to invokes the request pipeline using PipelineBuilder and Microsoft.Extensions.DependencyInjection.ServiceCollection. To utilize the module framework see BaseModule<TCommander>

Examples

The following code shows, how to utilize the TwitchCommander<TCommander>.

public class DummyCommander : TwitchCommander{T}
{
public DummyCommander(TwitchClient client) : base(client, "!")
{
}

public async Task InitializeAsync()
{
       await InitializeCommanderAsync(
      serviceCollection: BuildServiceCollection(),
    typeof(Program).Assembly,
    BuildRequestPipeline()
);
}

protected override async Task HandleCommandRequest(IResponse command)
{
 await base.HandleCommandRequest(command);
}

private static PipelineBuilder BuildRequestPipeline()
 => new PipelineBuilder()
      .UseMiddleware{TMiddleware}();

private static IServiceCollection BuildServiceCollection()
 => new ServiceCollection().AddSingleton{TService}();
}

HandleCommandRequest(IResponse)

HandleCommandRequest(IResponse) is a virtual base method that acts as event handler for all incoming message events. Overriding HandleCommandRequest(IResponse) allows you to add custom behavior before the request pipeline gets executed.

Important

IMPORTANT: not calling base.HandleCommandRequest(command) will lead to the request pipeline not getting executed!
